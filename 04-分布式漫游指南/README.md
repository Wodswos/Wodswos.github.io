# Introduction

不同的分布式场景有不同的条件和要求，因此并没有统一的方案。

甚至不同的方案可能从表面看可能风马牛不相及。

* 区块链

区块链的分布式网络中可能会存在恶意节点。

因此需要算力证明等机制增加恶意节点的成本，甚至使之在实际上难以成为现实。

* 分布式集群——存储or计算

往往不存在恶意节点，都是自己人，一般只需考虑宕机和网络通信异常。



* zookeeper集群



**分布式计算和存储说的是分布式的作用**
**而分布式通信却是分布式计算和存储时面临的问题**



如果存在完美的通信，比如彼此之前之间通信速度仅仅被物理层面的光速限制，且通信不会损坏，那么目前水平的分布式将变得再简单不过。（不过就算光速，在地球两端也有几十毫秒的误差）。
