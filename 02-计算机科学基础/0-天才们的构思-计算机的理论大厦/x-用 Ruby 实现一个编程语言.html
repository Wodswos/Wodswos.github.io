    用 Ruby 实现一个编程语言有一个缺点：这是在使用一种更复杂的语言解释执行一种简单的语言，从哲学上来说这可能很失败。

    但可以跟着书本《计算的本质：深入剖析程序和计算机》一步步用 Ruby 实现上述规则。

    Simple 语言：我们正在定义语义的语言

    Ruby 元语言：用来定义语义的语言

    操作语义

    从基础的算术开始，先定义数和加减法 

    class Number < Struct.new(:value)
    def to_s
        value.to_s
    end

    def inspect
        "<#{self}>"
    end
end

class Add < Struct.new(:left, :right)
    def to_s
        "#{left} + #{right}"
    end

    def inspect
        "<#{self}>"
    end
end

class Multiply < Struct.new(:left, :right)
    def to_s
        "#{left} * #{right}"
    end

    def inspect
        "<#{self}>"
    end
end

    表达式和规约

    • 什么样的表达式能规约： Add 和 Multiply 表达式总是能规约的， Number 表达式总是代表一个值，它就不能规约成任何其他东西了
    • 对能规约的表达式实现规约方法： Add 是加法， Multiply 是乘法
    • 如果加法左边的参数能够规约，就规约左边的参数，右边同理，如果都不能规约，将他们加到一起得到结果。乘法同理。
      • 显然，规约过程类似于一个深度优先遍历
      • 方法 reduce 把一个表达式转换为另一个表达式，但注意是构建出新的表达式，而不是对已有的表达式进行修改

    代码如下（Number 类的代码略） 

    class Add < Struct.new(:left, :right)
    def to_s
        "#{left} + #{right}"
    end

    def inspect
        "<#{self}>"
    end

    def reducible?
        true
    end

    def reduce
        if left.reducible?
            Add.new(left.reduce, right)
        elsif right.reducible?
            Add.new(left, right.reduce)
        else
            Number.new(left.value + right.value)
        end
    end

end

class Multiply < Struct.new(:left, :right)
    def to_s
        "#{left} * #{right}"
    end

    def inspect
        "<#{self}>"
    end

    def reducible?
        true
    end

    def reduce
        if left.reducible?
            Multiply.new(left.reduce, right)
        elsif right.reducible?
            Multiply.new(left, right.reduce)
        else
            Number.new(left.value * right.value)
        end
    end

end

# 测试用例
# 当前还没有语法解析器，所以 AST 要自己构建，注意先乘除，后加减。
expression = Add.new(
    Multiply.new(Number.new(1), Number.new(2)),
    Multiply.new(Number.new(3), Number.new(4))
)

while expression.reducible?
    puts expression
    expression = expression.reduce
end

puts expression

    上述测试代码可以抽象成一个 计算机器 ： 

    class Machine < Struct.new(:expression)
    def step
        self.expression = self.expression.reduce
    end

    def run
        while self.expression.reducible?
            puts expression
            step
        end
        puts expression
    end
end

expression = Add.new(
    Multiply.new(Number.new(1), Number.new(2)),
    Multiply.new(Number.new(3), Number.new(4))
)
my_machine = Machine.new(expression)
my_machine.run

    规约规则可以进一步丰富，如减法和除法、布尔值及其运算、数的比较等

    通过实现能对一种语言求值的虚拟机来定义它的操作语义。 虚拟机当前的状态就是当前的表达式 ，而机器的行为是由一个规则集合来描述的，这个规则集合负责管理及其运行时的状态切换。

    变量和环境

    可以用散列表存储变量名称到值的映射。 

    class Variable < Struct.new(:name)
    def to_s
        name.to_s
    end

    def inspect
        "<#{self}>"
    end

    def reducible?
        true
    end

    def reduce(environment)
        environment[name]
    end
end

    在前面所有关于 reduce 方法的定义和调用中，也应该都要加上参数 enviroment ，以确保 Machine 正确执行不会出错。
