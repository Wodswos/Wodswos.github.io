    排序的基本概念

    在 \(n!\) 中可能的排列情况中找到/确定唯一的顺序（降序 or 升序 or 自定义的序）排列。

    常用排序方法

    初级排序算法

    冒泡排序移动数组中的无序部分，依次将 无序部分中的最值移动到有序部分的末尾 ；插入排序移动数组中的有序部分，依次将 无序部分中任意的一个移动到有序部分中对应的位置 。

    冒泡排序和插入排序不仅时间复杂度高，还要移动（或者说交换）很多次。选择排序更像是冒泡的一丢丢优化版，通过一个 临时变量（或者说指针）记录最大值（或最小值） ，每次都与最大值比较，而不是带着最大值一路移动，以此减少交换次数。

    希尔排序

    h 有序数组：数组中任意间隔为 h 的元素都是有序的，即 h 个互相独立的有序数组编织在一起组成的数组。

    h 序列：在排序算法中，h 按照序列递减。显然，对于任意以 1 结尾的 h 序列，最终排序效果就是对整个数组的完整排序。 

    def shellSort(array):
    h = 1
    
    while h < len(array):
        h= 3*h + 1
    print(h)
    while h>= 1:
        # 对每一个数进行插入排序，不过只需保证其在 h 有序数组中有序，故在内层的循环中把 step 换为 -h
        # 这里第一个 for 也可以直接写 for i in range(N)，无所吊谓
        for i in range(h, len(array)):
            for j in range(i, h - 1, -h):
                if array[j - h] > array[j]:
                    array[j-h],  array[j] = array[j], array[j-h]
        h = h // 3
    return array

    查找的基本概念

    静态和动态查找

    • 静态查找：数据集合相对来说比较稳定，不需要进行插入和删除；
    • 动态查找：在查找的过程中同时需要插入和删除的操作；

    内查找和外查找

    • 在内存中进行的是内查找；
    • 在外存（硬盘）中进行的是外查找；

    平均查找长度

    ASL，Average Search Length

    其中 \(P_i\) 是查询第 \(i\) 个元素的概率，同常认为所有元素同概率，即 \(P_i = \frac 1 n\) ； \(C_i\) 是查询第 \(i\) 个元素所花费的查询次数。 \[ ASL = \sum _{i=1} ^N P_i \cdot C_i \]

def binarySearch (arr, x): 
    head = 0
    tail = len(arr) - 1
    while head < tail:
        mid = head + (tail - head)//2
        if arr[mid] == x: 
            return mid 
        elif arr[mid] > x: 
            tail = mid - 1
        else: 
            head mid + 1 
    return False

    LeetCode Q4: 求两个有序数组的总的中位数

    个人第一反应感觉这不像是二分查找的衍生啊？

    传统的二分查找，利用 一个 数组的有序性，找 任意一个数 ；而这一题，要求利用 两个 数组的有序性，找 特定的中位数 。这真的是用同一个逻辑吗？

    然后看到了官方的图（如下所示），我才恍然大悟， 题目要找的中位数只是表象，实际要找的是一根折线！ 如此一来，题目也确实变成了一个简单的二分查找。

    当然，官方的题解也说要确定的是红线的位置，中位数只与红线两侧的元素相关。

    但我第一遍看题解的时候（或者说看文字题解的时候），注意力还是下意识在元素上，而没有放到折线上，难免就感觉解法太过复杂，不得窥见其真谛。

    不知道有没有一般或者说具有普适性的方法洞察到这样一条折线，也许确实需要一点思想的火花？

    即 0 - 4 之间进行二分查找，不过判断条件不是二分查找原来简单的 \(a[mid] == key\) 了，而要稍微复杂一丢丢而已。

    实际复杂度还低于题目所说的 \(\log (m +n)\) （所以题目给的复杂度算不算一种误导？让人往同时处理两个数组的方向想？），是 \(\log (m+1)\) （如果 m 比 n 大，还可以交换一下两个数组，或者把 nums2 数组作为基准。）
