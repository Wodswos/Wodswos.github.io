    并查集 Union-Find

    数学原型：等价关系与划分

    所谓 等价关系，就是指复合自反性、对称性、传递性的关系。

    • 自反性：p 和 p 自身是相连的
    • 对称性：如果 p 和 q 是相连的，那么 q 和 p 也是相连的
    • 传递性：如果 p 和 q 是相连的且 q 和 r 是相连的，那么 p 和 r 也是相连的。

    就像偏序关系 = 自反 + 反对称 + 传递

    等价关系能将集合分为多个等价类。

    具体来讲，等价关系的现实情景可以有（详见《算法》第四版，Robert Sedgewick 著，谢路云 译，第1.5节）：

    • 网络：对于 p 和 q 之间是否需要假设一条新的连接才能通信
    • 变量名等价性：某些编程环境允许声明两个等价的变量名（多个引用指向同一对象），系统需要判别给定的变量名是否等价

    并查集实现

    我们在设计时面对的第一个任务就是精确地定义问题。我们希望算法解决的问题越大，他完成任务所需的时间和空间可能就越多。

    我们不可能预先知道这其间的量化关系，而且我们通常只会在发现解决问题很困难，或是代价巨大，或是在幸运地发现算法所提供的信息比原问题所需要的更加有用时修改问题。

    ——《算法》Robert Sedgewick 著 第四版

    并查集的应用也同样源于对问题的精确定义：只在乎特定的整数对 p q 是否在一个等价类（或者具体到某一问题中为是否相连等），而不在乎两者之间的通路上的所有连接。

    并查集支持两种操作：

    • 合并（Union）：把两个不相交的集合合并为一个集合
    • 查询（Find）：查询两个元素是否在同一个集合中

    quick-find

    即用一个数组，记录每一个点所属的等价类。 find() 操作只需 return id[p]==[q] ， union() 操作需要遍历数组找到某一个等价类的元素并复制。

    即这种方法 find() 操作的复杂度只有 \(O(1)\) ，但 union() 操作的复杂度会达到 \(O(N)\)

    quick-find 算法的运行时间对于最终只能得到少数连通分量的一般应用是平方级别的。

    quick-union

    与 quick-find 相似，同样用一个数组 id[] 记录信息，但不是直接记录的每个点所属的等价类。而是先将同一个等价类里的元素组织成树状的结构，用数组 id[q] 记录该点的父节点。（ id[] 数组用父链接的形式表示了一片森林。）

    union() 操作：找到其中一个等价类的根节点，将该根节点指向另一个等价类的根节点即可。

    find() 操作：不断递归，直至找到对应根节点即可。

    quick-union 的算法复杂度并不稳定，取决于树的高度的变化情况：在最好的情况下，可能达到线性级别，即接近 \(O(N)\) ；但在最坏的情况下，运行时间会达到 平方级别，即接近 \(O(N^2)\)

    加权 quick-union

    幸运的是，只需做一点简单的加法， quick-union 即可保证最坏的情况不会出现。

    用一个额外的数组 int[] sz 记录根节点对应等价类的大小（实际上是树的高度），从而实现尽可能把简单（实际上是较低）的树往复杂（较高）的树上合并（因为将较低的树合并到较低的树上，会使得最终的高度是更高的树的高度 + 1）

    路径压缩

    在查找算法中添加一点点小技巧，在最优的情况下，这行代码可能是多余的，甚至会浪费一丢丢时间，但在一般情况下，会极大地提高效率。 

    int find(int x)
{
    return x == fa[x] ? x : (fa[x] = find(fa[x]));
}
