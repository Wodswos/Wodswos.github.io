    最严格的语言可以使用形式语义学中的数学方法准确描述编程语言的含义。

    次之，如最常见的语言 C++、Java、EMACScript 等都有一份平实的官方文档，由专家委员会完成标准化，与具体实现无关。

    再次之，许多语言都没有官方的书面规范，往往以编译器或解释器实现为准。

    对于 Ruby 而言，尽管有很多关于 Ruby 应该如何工作的书和教程，但这些 资料的最终源头都是松本行弘先生的 Ruby 解释器（Matz’s Ruby Interpreter，MRI）。

    如果任何一份 Ruby 文档与 MRI 的实际行为不一致，那必然是文档错了。同理 Jruby，Rubinius，MacRuby 等第三方实现都只能努力地精准模拟 MRI 的行为，唯有如此，才算与 Ruby 语言有效兼容。

    （Ruby也有规范，甚至接受为 ISO 标准，但 MRI 依旧更为权威。）

    基础知识

    基本数据类型和数据结构

    ruby 仅允许通过赋值声明局部变量，如声明一个字符串 greeting = 'hello' ，而不能像 C++ 那样只声明，先不赋值。

    可以用一个数组给多个变量同时进行赋值，类似 python。

    ruby 符号

    在各种语言里，字符串很常见、很好理解，变量名也很常见、很好理解。而在 ruby 里还要一种奇奇怪怪的东西：符号。

    符号（symbol）是 ruby 内置类 Symbol 的实例。它有一个字面构造器：冒号引导符。 也可以通过这个冒号引导符，从字面上辨认出符号与字符串、变量名、方法或者其他的区别。

    对于字符串（string），可以调用 to_sym 方法转换为符号（symbol）；符号（symbol）也可以调用 to_s 方法转换为字符串（string）。

    字符串和符号的不同：

    • 对于给定的文本，只存在一个符号对象 ，即不管在代码的哪里重复声明同文本的符号，都是引用内存里的同一块内容。
    • 符号是不可变的，不能对符号的进行增、删、改。
    • 由于以上特性，符号是Ruby内部用来存取标识的系统元素

    匿名函数 proc

    一个 proc 是一段未经求值的 ruby 代码，其他语言把这种语言结构称为“匿名函数”或“lambda函数”。 

    multiply =-> x,y {x * y}
multiply.call(6, 9)

    ruby 中的对象和方法

    Ruby 的每个值都是一个对象，而且对象彼此之间靠发送消息进行通信。 每个对象都有自己的方法集合，这些方法决定了它如何响应特定的消息。

    甚至 1+2 都意味着“使用参数 2 给对象 1 发送一个叫做 + 的消息”，而对象 1 有一个处理该消息的方法 #+

    在所有的方法定义之外，当前对象是一个叫 main 的特殊顶层对象，任何没有指明接收者的消息都会被发送给它；同样，没有任何知名对象的方法定义都可以通过 main 使用。

    类和模块

    类 使得在许多对象之间共享方法定义成为可能。

    另一种共享方法定义的方式是在 模块（module） 中声明， 模块可以被任意类包括进去 ： 

    module Addition
    def add(x,y)
        x + y
    end
end

class AddingCalculator
    include Addition
end

    我们可以随时给类或模块增加方法，这种强大的特性称为 Monkey Patching

    枚举类型

    Enumerable 是 Ruby 的一个内置模块，数组、散列表等其他表示值的集合默认包含该模块。

    该模块中很多方法在调用时都可以带上一个代码块。 

    (1..10),count( |number| number.even?)
# 输出为 5，只有偶数被统计了

    结构体

    结构体（Struct）是 ruby 中一个特殊的类（Class），它的工作是生成其他类。根据传进 Struct.new 的每个属性名，Struct 产生的类会包含相应的获取方法和设置方法。

    要使用由结构体生成的类，常见方式是对其进行子类化 

    class Point < Struct.new(:x, :y)
    def +(other_point)
        Point.new(x + other_point.x, y + other_point.y)
    end
    
    def inspect
        "#<Point (#{x}, #{y})>"
    end
end

    一些特性

    • 任何以大写字母开头的变量都是常量，可以重新赋值，但 Ruby 会产生警告
    • 一个顶层常量可以通过给 Object 发送消息 remove_const 来删除
      • Object.send(:remove_const, :const_name)
      • 但不能用 Object.remove_const 的方式来调用，因为该方法是一个私有方法，只能通过 send 方法进入类内部发送消息调用，从而跳过这个限制
    • 和 python 类似，ruby 也支持可变参数。但与 python 不同的是，ruby 中常规参数放到可变参数的前后都可以。

    输出显示和字符串操作

    • 用 #{var_name} 可以实现在字符串内插值， 不过要注意，此时字符串得是双引号的 （这个好像跟 bash 类似），即与 python 不同，ruby 中单引号和双引号的字符串不等价
    • inspect 多用于调试的输出，而 to_s 则用于最终用户或者显示的目的.
      • 当一个对象被当作字符串操作时，默认会调用其 to_s 方法，类似 Java 的 toString()
      • 每当 IRB 需要显示一个对象，都会向这个对象发送 inspect 消息（即调用对象的 inspect 方法），然后这个对象返回自身的字符串表示。
    • puts 方法可以用来向标准输出字符串。该方法对每个 ruby 对象可用（ puts self 会得到 main ），非 String 则输出该对象 to_s 方法的内容。

    代码块

    代码块是由 do/end 或者大括号围住的一段 Ruby 代码。代码块可以带参数。 

    # 其中 n 就是参数
{ |n| puts "#{n}, hello"}

    对于形如 {object} object.method 的代码块，有更简洁的写法： &:method

    对象的方法（函数）可以带一个隐式的代码块参数，并用 yield 关键字表示调用这段隐式的代码块。 

    def do_three_times
    yield
    yield
    yield
end

do_three_times {puts 'hello'}

# 输出：
# hello
# hello
# hello
