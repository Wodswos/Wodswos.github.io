    选取（定位）正确的元素

    设置 style 的三种方式

    • 内联样式
      • 在每个元素标签内，直接通过 style 属性指定其他属性。
      • <h1 style="color: blue;background-color: yellow;border: 1px solid black;">Hello World!</h1>
      • 这种方式网站稍有规模便很难维护
    • 内部样式表
      • 在 <head> 标签里添加 <style> 标签，在 style 标签内通过选择器选择元素，从而指定属性。
    • 外部样式表
      • <link rel="stylesheet" href="styles/general/style.css">

    选择器

    最基本的选择器（或者说选择条件）有如下：

    选择器名称 选择的内容 示例
    元素选择器（也称作标签或类型选择器） 所有指定 (该) 类型的 HTML 元素 p 选择 <p>
    ID 选择器 具有特定 ID 的元素（单一 HTML 页面中，每个 ID 只对应一个元素，一个元素只对应一个 ID） #my-id 选择 <p id="my-id"> 或 <a id="my-id">
    类选择器 具有特定类的元素（单一页面中，一个类可以有多个实例） .my-class 选择 <p class="my-class"> 和 <a class="my-class">
    属性选择器 拥有特定属性的元素 img[src] 选择 <img src="myimage.png"> 而不是 <img>
    伪（Pseudo）类选择器 特定状态下的特定元素（比如鼠标指针悬停） a:hover 仅在鼠标指针悬停在链接上时选择 <a> 。

    可以在以上基本选择器的基础上进行一些简单组合，主要有：

    • condition1,condition2 ：逗号连接两个选择条件，表示满足 condition1 或 满足 condition2 的元素
    • condition1.condition2 ：句点连接两个选择条件，表示满足 condition1 且 满足 condtion2 的元素
    • condition1 condition2 ：空格连接两个选择条件，表示满足 condition1 的元素内的所有满足 condition2 的元素
    • condition1>condition2 ：父级元素满足 condition1 、且自身满足 condition2 的元素
    • condition1+condition2 ：紧接着满足 condition1 的元素、且自身满足 condition2 的元素

    选择符当然还可以进一步任意组合，如 condition1 condition2 condition3 表示满足 condition2 的元素内的所有满足 condition3 的元素，而该满足 condition2 的元素在满足 condition1 的元素内。还可以更复杂一些，如 body h1 + p .special

    关于属性选择器：

    • [attribute]
    • [attribute=value]
    • [attribute~=value]

    特殊地， * 表示选择所有元素

    nth-of-type() 和 nth-child()

    级联（层叠）与继承

    继承

    继承指的是类似 color , font-family , visibility 等属性父元素设置，子元素会被继承的特性。

    像 width （上面提到的）、 margin 、 padding 和 border 不会被继承。

    控制继承

    CSS 为控制继承提供了五个特殊的通用属性值。每个 CSS 属性都接收这些值。

    inherit 、 initial 、 revert 、 revert-layer 、 unset 

    .my-class a {
    color: initial;
}

    级联（层叠，cascade）

    级联（cascade）在计算机科学里指多个对象之间的映射关系，建立数据之间的级联提高管理效率。

    重复性的操作十分繁琐，尤其是在处理多个彼此关联对象情况下，此时我们可以使用级联（cascade）操作。级联在关联映射中是个重要的概念，指当主动方对象执行操作时，被关联对象（被动方）是否同步执行同一操作。

    级联还指用来设计一对多关系。

    在电气系统中，把二个以上的设备通过某种方式连接起来，能起到扩容的效果就是级联。还比如网络上的交换机、路由器级联。

    如下图所示，在浏览器的开发者模式中，选中任一元素，可以查看 施加在该元素上的层叠样式代码 ，在上面的样式具有更高的优先级（如该 <h1> 标签继承自 <body> 的 font-family 属性被开发者用标签选择器指定的 font-family 所覆盖。）

    虽然这个例子里准确来说这是继承被覆盖，而不是级联？又或者说继承就是最低优先级的级联。

    你甚至可以在开发者模式直接新增、更改这些属性，浏览器会返回新的渲染结果。

    级联规则（优先级，specificity）

    以下内容 copy 自：

    https://www.zhangxinxu.com/wordpress/2022/05/deep-in-css-cascade/

    如上图所示（上图中未包括 transition 和 animation），最终应该用哪个样式，是由级联规则来决定的， 一般来说，越具体优先级越高，这也比较符合常识。

    相互冲突的声明将按以下顺序（也即上图）应用，后一种声明将覆盖前一种声明：

    1. 用户代理样式表中的声明（例如，浏览器的默认样式，在没有设置其他样式时使用）。
    2. 用户样式表中的常规声明（由用户设置的自定义样式）。
    3. 作者样式表中的常规声明（这些是我们 web 开发人员设置的样式）。
    4. 作者样式表中的 !important 声明
    5. 用户样式表中的 !important 声明
    6. 用户代理样式表中的 !important 声明

    同级的时候看先后顺序，后来者优先级更高，即前者被覆盖。

    还有 animation 和 transition 等，但具体到浏览器，可能会和规范有所差异，故作为一个外行人就不去多做了解了。

    浏览器内置和用户设置

    很多 HTML 元素会内置样式，常见的如 <body> , <p> 等元素的 margin 外间距， <input> 、 <button> 等元素的边框和背景色样式等。

    用户设置的 CSS 多指用户安装插件等工具所设置的 CSS（称为“注入的样式表”），比方说 Adblock 等插件对页面元素隐藏的 CSS 就属于这一类。

    （不准确地来说）用户通过浏览器自身的设置改变浏览器的主题背景，字号或字体等也可以归为这一类。

    @layer 规则

    现代浏览器纷纷支持了一个极具颠覆性的新特性，即 @layer ，这个规则可以让其中设置的所有 CSS 的优先级降低，特别适合在引入某个模块的时候使用，可以优先避免他人的代码因为优先级原因影响自己的代码。 

    button {
  color: red;
}
@layer {
  #button {
    color: green;
  }
}

    上述代码中，尽管 ID 选择器的优先级更高，但由于 @layer 规则的存在，最终 button 的 color 还是 red。

    网页开发者设置

    !important

    !important 有个很有意思的特性，那就是原本级联水平高的CSS声明应用了 !important 后，其优先级反而低，而原本级联水平低的 CSS 声明应用了 !important 后，CSS 计算的优先级反而高。

    例如，用户代理 CSS（浏览器内置 CSS）在级联规则中处于底层，但是，一旦这些属性值的后面增加了 !important 就几乎不可撼动。

    案例：Chrome 广告拦截插件，通过 !important 设置样式，使得广告对应的容器隐藏，此时网站开发者即使同样加 !important 也优先级不如，故难以越过插件让广告展示出来。

    盒子模型和 Normal Flow

    一般每个块级元素都有三个盒子层层包裹。

    • Content
      • 盒子模型的最内层。
      • 默认情况下，当指定元素的 width 和 height 属性时，设置的就是 content 的大小
    • 内边距，Padding
      • padding 是 元素内部空出的空间 ，该部分的颜色由属性 background-color 的决定，而不能像 border 一样指定。
      • padding 不能使用负值
    • 边框，Border
      • border 是边框， 可以设置厚度、颜色、样式（solid、dashed……）等。
    • 外边距，Margin
      • 盒子模型的最外层， 可以是负值
      • margin 同样不能设置颜色，是完全透明的。
      • 外边距折叠：两个相邻的元素之间的边距并不是两者外边距的和，而是两者中的较大者（两个外边距合并为一个外边距）
      • margin 设置为 0 auto 可以起到左右居中的效果，其中 0 表示上下的 margin 为 0（等效 0 auto 0 auto ）。

    对于 width 和 height 定义为 content 的长宽这一点，如果不喜欢这种对定义，觉得 padding 什么的也应该包括在长宽里，则可以使用替代盒模型（设置属性 box-sizing: border-box ），在替代盒模型中， width 和 height 是以 border 为界的总长宽

    如下代码可以实现整个 html 都用替代盒模型，而不是标准盒模型 

    html {
  box-sizing: border-box;
}
*, *::before, *::after {
  box-sizing: inherit;
}

    块级盒子 vs 内联盒子

    块级盒子（如 <div> 、 <h1> 、 <p> 等都是块级元素）：

    • 盒子会在内联的方向上扩展并占据父容器在该方向上的所有可用空间，在绝大数情况下意味着盒子会和父容器一样宽
    • 每个盒子都会换行
    • width 和 height 属性可以发挥作用
    • 内边距（padding）, 外边距（margin）和 边框（border）会将其他元素从当前盒子周围“推开”

    内联盒子（如 <a> 、 <span> 、 <em> 以及 <strong> 都是默认处于 inline 状态的。）：

    • 盒子不会产生换行。
    • width 和 height 属性将不起作用。
    • 垂直方向的内边距、外边距以及边框会被应用但是不会把其他处于 inline 状态的盒子推开。
    • 水平方向的内边距、外边距以及边框会被应用且会把其他处于 inline 状态的盒子推开。

    display 属性和 inline-block

    可以通过对盒子 display 属性的设置，比如 inline 或者 block ，来控制盒子的外部显示类型。

    英文段落显示在一个段落的下面，这是因为它们的样式是 display:block 。如果在段落中的某个文本周围创建链接，则该链接将与文本的其余部分保持内联，并且不会打断到新行。这是因为 <a> 元素默认为 display:inline 。

    inline-block 在内联和块之间提供了一个中间状态。

    • 设置 width 和 height 属性会生效。
    • padding , margin , 以及 border 会推开其他元素。

    但是，它不会跳转到新行，如果显式添加 width 和 height 属性，它只会变得比其内容更大。

    正常布局流 Normal Flow

    也称文档流。

    display:flex 和弹性布局

    flexible box 通常也称为 flexbox 或 flex，也称为弹性盒子或弹性布局。一种比较高效率的 css3 布局方案。

    flex 有两根轴线，分别是主轴和交叉轴，主轴的方向由 flex-diretion 属性控制，交叉轴始终垂直于主轴。

    • flex容器（flex container） 是设置了 display: flex 的父元素。
    • flex项（flex item） 是指在 flex 容器中表现为弹性的盒子的元素。
    • 主轴 （main axis）是沿着 flex 元素放置的方向延伸的轴（比如页面上的横向的行、纵向的列）。
      • 其开始和结束被称为 main start 和 main end 。
      • flex-direction 属性可以指定主轴的方向，初值为 row ，故子元素（flex项）会整整齐齐排成一行，除此以外还有 column 、 row-reverse 等
    • 交叉轴 （cross axis）是垂直于 flex 元素放置方向的轴。
      • 其开始和结束被称为 cross start 和 cross end 。

    flex容器的属性

    • flex-flow 属性（缩写属性）
      • flex-flow 是 flex-direction 和 flex-wrap 两个属性的缩写。
      • flex-direction ：定义了主轴的方向，默认 row 。
      • flex-wrap ：指定 flex 元素单行显示还是多行显示。默认 nowrap ，子元素被摆放到一行。
    • align-items 属性
      • 默认值是 stretch ，即所有子元素沿着交叉轴方向拉伸至和最高的元素保持同步
      • center 会让所有子元素保持各自的高度，并在交叉轴方向保持居中
      • flex-start 和 flex-end 表示在交叉轴的开始或结束处对齐
      • 可以在子元素中用 align-self 属性覆盖容器带有的 align-items 属性
    • justify-content 属性
      • 控制 flex 项在主轴上的位置，默认值 flex-start
      • space-around 使所有 flex 项沿着主轴均匀地分布，在任意一端都会留有一点空间。

    flex项的属性

    • flex 属性（缩写属性）
      • 属性的值可以是 无单位的比例值 。将所有的子元素赋值为同等值（如 1 ），则所有的子元素都会等宽地伸展并填充容器，不会在尾部留下空白。
        • 实际上此时指定的是 flex-grow 属性
      • 也可以是固定的宽度， 指定 flex 的最小值 ，如 200px ，则每个项将 至少为 200px 宽 。
        • 实际上此时指定的是 flex-basis 属性
        • 当 flex-wrap 值为 warp 时，每行（ flex-direction 方向）都会装入尽可能多的 flex 项，然后换行继续装。
      • flex 作为缩写属性自然可以同时指定两种值，如 flex: 1 200px;
        • 此时会先分出固定数值的空间（即 basis 的意思），然后将剩余空间按比例分配（即 grow 的意思）。
        • 除此以外缩写属性 flex 还包括 flex-shrink 属性
    • order 属性
      • 可以给 flex 项赋值进行排序， 默认值是 0 。
      • 可以设置负值使得对应元素比默认元素排序更前。

    定位和 position 属性

    如果你想要创建一个浮动在页面其他部分顶部的 UI 元素，并且/或者始终停留在浏览器窗口内的相同位置，无论页面滚动多少？定位使这种布局工作成为可能。

    • 静态定位： static ，元素的 position 属性的默认值，即将元素放在文档布局流的默认位置
    • 相对定位： relative ，相对于元素在正常的文档流中的位置移动它——包括将两个元素叠放在页面上。
      • left:20px 可以理解为一个从左边来的力推动这个 block 移动 20 px，故实际上是向右偏移；
      • top 、 bottom 、 right 同理
    • 绝对定位： absolute ，将元素完全从页面的正常布局流（normal layout flow）中移出，类似将它单独放在一个图层中。
      • 内容较多，详见后文
    • 固定定位： fixed ，与绝对定位非常类似，但是是将一个元素相对浏览器视口固定，而不是相对另外一个元素。
    • 粘性定位： sticky ，一种新的定位方式，它会让元素先保持和 position: static 一样的定位，当它的相对视口位置（offset from the viewport）达到某一个预设值时，它就会像 position: fixed 一样定位。

    绝对定位

    绝对定位的元素不再存在于正常文档布局流中，而相对定位并不会删除对应元素在文档流中占据的空间

    在说绝对定位怎么生效和使用之前，需要引入一下包含块（containing block）的概念，因为离开这个概念会很难理解绝对定位的偏移值。

    在 Normal Flow 中，如果有一个 <div> 或 <p> 这样的元素如果有 margin-top 这样的属性，并且值为30%时，显然其意思是包含他的那个容器的 30%。

    但绝对定位的元素脱离了 Normal Flow ，又该如何处理类似 30% 这种值呢。

    包含块（containing block）

    大多数情况下，包含块就是这个元素最近的祖先 block box 的 content 部分，但也不是总是这样。

    确定一个元素的包含块：

    1. 如果 position 为 static 、 relative 或 sticky ，此时为 normal flow，故包含块一般由它的最近的祖先 块元素 的内容区的边框组成。（也可能会建立格式化上下文）
    2. 如果 position 为 absolute ，包含块就是由离它的最近的、 position 的值不是 static 的祖先元素的内边距区的边缘组成。
    3. 如果 position 为 fixed ，在连续媒体的情况下 (continuous media) 包含块是viewport，在分页媒体 (paged media) 下的情况下包含块是分页区域 (page area)。
    4. 更多内容略，详见https://developer.mozilla.org/zh-CN/docs/Web/CSS/Containing_block#identifying_the_containing_block

    如果声明了 absolute 的元素的所有的父元素都没有显式地定义 position 属性，那么所有的父元素默认情况下 position 属性都是 static，此时就需要 初始块容器 登场。

    初始块容器 ：其有着和浏览器视口一样的尺寸，并且 <html> 元素也被包含在这个容器里面。

    简单来说，绝对定位元素会被放在 <html> 元素的外面，并且根据浏览器视口来定位。

    如此一来，就可以根据 containing block 来计算 height 、 width 等属性中的百分值了。

    此时 top 指定该元素顶部距离 containing block 顶部的距离， bottom 、 right 、 left 、 margin 等属性同理

    z-index 属性

    元素开始重叠（就像多个图层），故我们还需要设置元素（图层）哪些在更上层，哪些在更下层。

    正值将它们移动到堆栈上方，负值将它们向下移动到堆栈中。默认情况下，定位的元素都具有 z-index 为 auto，实际上为 0。

    固定定位

    这与绝对定位的工作方式完全相同，只有一个主要区别：绝对定位将元素固定在相对于其位置最近的祖先。（如果没有，则为初始包含它的块）而固定定位固定元素则是相对于浏览器视口本身。这意味着你可以创建固定的有用的 UI 项目，如持久导航菜单。

    粘性定位

    基本上是相对位置和固定位置的混合体，它允许被定位的元素表现得像相对定位一样，直到它滚动到某个阈值点（例如，从视口顶部起 10 像素）为止，此后它就变得固定了。

    背景和边框

    背景的属性可以是简单的 background: gray; ，也可以复杂如斯： 

    .box {
  background: linear-gradient(105deg, rgba(255,255,255,.2) 39%, rgba(51,56,57,1) 96%) center center / 400px 200px no-repeat,
  url(big-star.png) center no-repeat, rebeccapurple;
}

    background 也是简写属性，其可以的包括内容有：

    • backgournd-color
    • background-image （通常值为 url(xxx) ）等
      • 也可以在单个属性值中指定多个 background-image 值，用逗号分隔每个值。（背景图像会互相重叠）
    • background-repeat ：控制背景的平铺方式
    • background-size ：调整背景大小，可以直接设置长宽，也可以设置 cover 、 contain 等关键字
    • background-position
    • background-attachment ：控制背景如何随着页面的滚动变化。

    关于 CSS 中各路 color 的值，可以是如 #ffffff 表示的 RGB 三元组，也可以是常用颜色如红色的英文拼写 red ，也可以是 RGBA （A 为透明度，介于 0~1 之间）的四元组 rgba(255, 255, 255, 0.5) 。

    有一些规则，需要在简写背景属性时遵循，例如：

    • background-color 只能在逗号之后指定。
    • background-size 值只能包含在背景位置之后，用’/’字符分隔，例如： center/80% 。

    对于 border ，除了最常用的 color 、 style 、 width 三个属性外，还有 border-radius （边框圆角化）等。

    调整容器尺寸

    固有尺寸：把图片放置在网页中的时候没有在 <img> 标签或 CSS 中设置其尺寸，那么将使用其固有尺寸显示。

    具体尺寸：参考前述长度属性及其单位。

    min- 和 max- 尺寸：元素的固有尺寸由其内容决定，而内容的量可以是 \((0,+\infty)\) ，故元素块可能出现过大或过小的极端情况。此时可以通过 min- （如 min-height ）和 max- 属性来为其进行一定的设限，这样一来，即使内容出现极端情况，元素的尺寸变化也不会超过设限的范围。

    对于图像而言， width 属性会拉伸原图像到充满容器，而 min-width 和 max-width 不会。这个技术是用来让图片 可响应 的，所以在更小的设备上浏览的时候，它们会合适地缩放。

    一个空的 <div> 是没有尺寸的，如果给这个空的 div 赋予一个边框，在网页中看到的实际效果会是上下边框二合一的一条线。

    元素的高度由其所含内容高度确定。再强调一次，这就是 元素的固有尺寸 — 由其所包含的内容决定。

    逻辑属性和逻辑值

    当我们给 <div> 设置 width 属性时， 我们希望的效果，未必是让 div 的宽度取这个值，而是希望每一行字（当方向为 vertical 时为每一列字）的长度是这个值，但浏览器只会理解成你是想让 div 的宽度取这个值 。显然此时我们需要一些全新的属性来避免这种误会： 逻辑属性（与 width 这种直观的物理属性相对） 。

    横向书写模式下，映射到 width 的属性被称作内联尺寸（ inline-size ）——内联维度的尺寸。而映射 height 的属性被称为块级尺寸（ block-size ），这是块级维度的尺寸。

    其他物理属性如 margin-top 、 padding-left 等也有相应的映射。如 margin-top 属性的映射是 margin-block-start ——总是指向块级维度开始处的边距。

    padding-left 属性映射到 padding-inline-start ，这是应用到内联开始方向。

    相应的，一些属性的取值是一些物理值（如 top 、 right 、 bottom 和 left ）。这些值同样拥有逻辑值映射（ block-start 、 inline-end 、 block-end 和 inline-start ）。

    简写属性（shorthand）

    简写属性 是将一些具有强共性的属性聚合在一起书写的属性，即可以让你同时设置好几个 CSS 属性的值。从而提升效率的同时能使代码（样式表）更整洁、更具有可读性。

    如常见的 font , background , padding , border , and margin 等属性： 

    padding: 10px 15px 15px 5px;
background: red url(bg-graphic.png) 10px 10px repeat-x fixed;
/*以上代码与以下代码等效 */
padding-top: 10px;
padding-right: 15px;
padding-bottom: 15px;
padding-left: 5px;
background-color: red;
background-image: url(bg-graphic.png);
background-position: 10px 10px;
background-repeat: repeat-x;
background-attachment: fixed;

    还有如 all 将除了 unicode-bidi 与 direction 之外的所有属性重设至其初始值，或继承值。
