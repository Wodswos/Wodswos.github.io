    蛮荒：Mosaic、Netscape

    1992年底， 伊利诺大学的NCSA组织认为，浏览器无图无真相，于是 Marc Andreessen 领导团队开发出第一个被广泛使用、支持图像显示的浏览器 Mosaic （马赛克），于1993年发布。

    Mosaic支持图片，但前辈Nexus不支持图片显示，网站开发者该如何平衡呢？UserAgent应运而生。

    通过UserAgent字段告诉服务器自己的浏览器类型，服务器可以据此决定HTML的具体内容。如果UserAgent为包括Mosaic则表示支持图片现实，服务器可以返回图片内容。

    Mosaic的中心人物在加州设立了“MOSIAIC Communication Corp”。但此时由伊利诺伊大学所拥有的Mosaic已经转让给了Spyglass Entertainment公司。

    开发团队从头开始攥写浏览器代码，1994年10月发布Mosaic Netscape 0.9取名Mozilla，后为了避免商标版权问题，公司改名为Netscape Communication Corporation。

    同年12月，该公司正式推出浏览器1.0，命名为Netscape Navigator。（内部代号：Mozilla）

    内部代号Mozilla更能体现出该浏览器的特质，一是谐音 Mosaic Killer ，二是谐音猛兽 Godzilla （哥斯拉）。

    NetScape上市第一天市值从28刀飙涨刀75刀（有点夸张，没具体考证，百度百科抄的）。1995年该公司的收入每季度翻一倍（跟它一比其他独角兽公司都不算啥了）。

    世界上第一款 JavaScript 引擎是伴随第一个能运行 JavaScript 程序的浏览器出现的，也就是 Netscape Navigator.

    很快，NetScape凭借着其更优的展示效果，战胜了Mosaic，成为了新的霸主。

    NetScape最先支持html框架显示，如简单的Table布局、内外边距等。

    虽然Mozilla改名为Netscape Navigator，但其UserAgent还是使用Mozilla，至今Mozilla也依旧作为UserAgent如雷贯耳。

    对于服务器而言，如果UserAgent中存在Mozilla，则表示可以发送带有HTML框架的页面。

    因为Netscape与Sun合作，Netscape管理层希望它外观看起来像Java，因此取名为JavaScript。但实际上它的语法风格与Self及Scheme较为接近。

    1994年，Opera作为一个研究项目启动。

    Opera是现存主要浏览器中发布最早的，却一路陪跑，没有淘汰，也没有成为领导者。就像自己开了一条支线。

    1996年，Opera 2.10发布。

    巨龙：IE

    之前提到Mosaic的版权由伊利诺伊大学转让给了Spyglass Entertainment公司，随后微软从Spyglass Entertainment公司手中买下Mosaic的授权。

    1995年，巨头微软基于Mosaic发布IE，凭借着和Windows的绑定销售，逐渐在市场份额上乱杀。

    1997年7月7日Mosaic停止开发。

    1997年，IE采用Trident引擎。

    有一段时间，Internet Explorer 6 垄断了市场，几乎没有人使用任何其他浏览器。但……微软在浏览器上的傲慢终究会带来报应。

    1998年，Netscape Navigator终究无法阻止拉下脸的巨头微软。随后网景公司宣布旗下所有软件免费，并开源了Netscape Communicator 4.0的代码。

    程序员传统艺能：开源

    Netscape Navigator成立了非正式组织Mozilla（与浏览器的内部代号同名）。组织开发出了更出色的新一代网络应用软件套装“Mozilla Application Suite”。

    它所采用的排版引擎被命名为“Gecko”。

    Mozilla组织声称他们主要开发用于测试目的的Mozilla浏览器，这个浏览器不适合直接为最终用户使用。

    Mozilla Develop Network

    主要内容包括 HTML, Java, CSS 以及其它项目内容的参考文档和教程，内容由社区开发者和技术文档工程师(Technical Writer)维护。很早以前便成为事实上的开发者 Web 技术参考平台。

    2017年10月18日， Mozilla 携手 Google、微软、三星、W3C 同时宣布将 MDN 作为 Web 文档的中心，共同为开发者创建一个统一的、跨浏览器的权威参考文档。微软宣布将超过 7700 个 MSDN 页面重定向到 MDN web docs ，Google 表示过去几年 Chrome 已将 Web 文档方面的努力转移到 MDN 。

    2002年，

    2003年，苹果发布Safari。

    在2007年之前，所有的浏览器都采用单进程架构。

    明面上，IE和Edge还有着极其可观的市场占有量，但大家都心知肚明，微软大势已去，不再是标准的制定者了。

    行业新标杆：Chromium

    Chrome 浏览器先使用 WebKit 引擎后使用 Blink 引擎。

    在现在的Chrome浏览器控制台输入 navigtor.userAgent 会返回类似如下内容：

    Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/88.0.4324.150 Safari/537.36

    从某种程度上UserAgent字段也是浏览器发展历史的一个见证者。

    在Edge浏览器中UserAgent还会有一条 Edg/88.0.705.68 ，在Opera浏览器中UserAgent还会有一条 OPR/74.0.3911.107 。但大家都有的同类项就是 Mozilla 、 AppleWebKit 和 Chrome 。

    2016年，Opera确定被奇虎360和昆仑万维收购。

    Chromium概述 3

    Chromium is an open-source browser project that aims to build a safer, faster, and more stable way for all users to experience the web. 4

    对于很多人而言，提到浏览器基本上就默认火狐、Chrome、Safari了，IE已经逐渐成为历史，Edge也难以力挽狂澜，虽然IE和Edge占了很大的市场，但显然标准已经不是巨硬说了算了。

    Chromium是Chrome的开源版，某种程度上的行业标准。

    Github上的Chromium Project 5 主要有如下目录 6 ：

    • base：通用代码，基础组件，包含字符串、文件、线程、消息队列等工具类集合
    • cc：Chromium Compositor，负责渲染合成
    • chorme：Chromium浏览器外壳实现
    • content：多进程沙盒浏览器的核心代码，管理进程架构和线程架构
    • v8：V8 JavaScript引擎库
      • 包括字节码解析器，JIT 编译器，多代GC，inspector (调试支持)，内存和 CPU 的 profiler（性能统计），WebAssembly 支持，两种 post-mortem diagnostics 的支持，启动快照，代码缓存、代码热点分析
      • 内容几乎覆盖整个《编译原理》
    • net：网络栈实现
      • 包含主机解析，cookies，网络改变探测，SSL，资源缓存，ftp，HTTP， OCSP实现，代理 (SOCKS和HTTP) 配置，解析，脚本获取（包括各种不同系统下实现），QUIC，socket池，SPDY，WebSockets等等等等。
      • 内容几乎覆盖整个《计算机网络》
    • ipc：进程间消息通信实现
    • mojo：类似于Android的AIDL，提供了跨语言（C++/Java/JavaScript）跨平台的进程间对象通信机制
    • media：多媒体封装代码，包含媒体内容捕获和播放的组件集合
    • gpu：OpenGL封装代码，包含CommandBuffer和OpenGL兼容性支持
    • skia：图形库，存放的是Chromium对skia的配置和扩展代码
      • 包括十几种矢量的绘制，文字绘制、GPU加速、矢量的指令录制以及回放（还要能支持线程安全）、各种图像格式的编解码……
      • 更多的，包括PDF的生成（Skia支持把矢量图绘制成pdf）、GPU渲染优化（即以上部分功能需要用gpu来渲染）
      • skia是Google收购的
    • third_party：网页排版引擎，第三方库
    • ui：UI框架

    还有一些其他次重要的目录：

    • android_webview：适合集成在安卓平台，但不适合单个apk使用
    • apps：Chrome打包的应用

    • android_webview: Provides a facade over src/content suitable for integration into the android platform. NOT intended for usage in individual android applications (APK). More information about the Android WebView source code organization.

    • apps : Chrome packaged apps .

    • base : Common code shared between all sub-projects. This contains things like string manipulation, generic utilities, etc. Add things here only if it must be shared between more than one other top-level project.

    • breakpad : Google’s open source crash reporting project. This is pulled directly from Google Code’s Subversion repository.

    • build : Build-related configuration shared by all projects.

    • cc : The Chromium compositor implementation.

    • chrome : The Chromium browser (see below).

    • chrome/test/data : Data files for running certain tests.

    • components : directory for components that have the Content Module as the uppermost layer they depend on.

    • content: The core code needed for a multi-process sandboxed browser (see below). More information about why we have separated out this code.

    • device: Cross-platform abstractions of common low-level hardware APIs.

    • net : The networking library developed for Chromium. This can be used separately from Chromium when running our simple test_shell in the webkit repository. See also chrome/common/net .

    • sandbox : The sandbox project which tries to prevent a hacked renderer from modifying the system.

    • skia + third_party/skia : Google’s Skia graphics library. Our additional classes in ui/gfx wrap Skia.

    • sql: Our wrap around sqlite.

    • testing : Contains Google’s open-sourced GTest code which we use for unit testing.

    • third_party : 200+ small and large “external” libraries such as image decoders, compression libraries and the web engine Blink (here because it inherits license limitations from WebKit). Adding new packages .

      • …/blink/renderer : The web engine responsible for turning HTML, CSS and scripts into paint commands and other state changes.

    • tools

    • ui/gfx : Shared graphics classes. These form the base of Chromium’s UI graphics.

    • ui/views : A simple framework for doing UI development, providing rendering, layout and event handling. Most of the browser UI is implemented in this system. This directory contains the base objects. Some more browser-specific objects are in chrome/browser/ui/views.

    • url : Google’s open source URL parsing and canonicalization library.

    • v8 : The V8 Javascript library. This is pulled directly from Google Code’s Subversion repository.

    Chromium主要分为两部分，即浏览器Browser和渲染器Render。

    Chromium is separated into two main parts (excluding other libraries): the browser and the renderer (which includes Blink, the web engine). 7

    The browser is the main process and represents all the UI and I/O. The renderer is the (often) per-tab sub-process that is driven by the browser. It embeds Blink to do layout and rendering.
