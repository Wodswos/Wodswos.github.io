    Github Pages

    （关于 git 的的架构和基本操作，可以参考另一篇笔记[[Git-项目协同的典范]]）

    官方的参考资源很丰富：

    quick start：https://pages.github.com/ ；https://docs.github.com/en/pages/quickstart

    Get Started：https://docs.github.com/en/pages/getting-started-with-github-pages/about-github-pages

    Examples：https://github.com/collections/github-pages-examples

    Set up site with Jekyll：https://docs.github.com/en/pages/setting-up-a-github-pages-site-with-jekyll/about-github-pages-and-jekyll

    有三种类型的 Pages Site：项目的、用户的、组织的。项目的 site 与某个 repo 相关，后两者与某个具体的账户相关（且需新建仓库并命名为 <username/orgnization>.github.io ）。后两者和账户唯一绑定，显然只能有一个，但项目性的 pages site 数量不限。

    GitHub Pages 不支持服务器端语言如 PHP, Ruby, or Python.

    GitHub Pages supports more than 750 MIME types across thousands of file extensions.

    Deploy from a Branch 即 Jekyll

    最基础的选择，不需要太多的操作，就可以完成简单但完整的部署流程。

    只需在 Repo 首页依次找到 Settings - Code and automation - Pages - Build and Deploymen - Source - Deploy from a branch - branch - main - root/docs - Save 即可。（通常项目的文档会在 /docs 目录下，而个人或组织的 Pages 会直接放根目录下）

    可以在 publishing source的根目录创建一个 .nojekyll 文件禁用 Jekyll 构造流程。
