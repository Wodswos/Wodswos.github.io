    Jekyll 和 hexo 简介

    Jekyll

    Jekyll 后面是两个小写的 L，而非大写的 i.

    以下内容转载自https://jimmysong.io/hugo-handbook/tools/jekyll.html

    Jekyll 使用 Ruby 语言开发，是最早开始流行的静态网站构建工具，至今依旧风靡，也是Github Pages默认的静态网站构建工具。

    Jekyll网站构建的理念就是只做用户告诉它要做的事情，不多也不少：

    1. No Magic ：简单易懂
    2. It “Just Works” ：初始化网站配置，不需要为让用户操心
    3. Content is King ：内容为王，这是用于应该真正关心的部分
    4. Stability ：向后兼容，容易维护
    5. Small & Extensible ：核心功能满足90%用户的需求，其它功能使用插件形式，易于集成

    以上理念也被后来出现的其它静态网站构建工具所传承。

    Hexo

    Hexo 是一款使用 node.js 开发的静态网站构建工具，便于构建华丽绚烂的页面。

    可处理的文件格式

    hugo 通过 front matter（如果有）或文件的扩展名来确定是否需要进行相应的处理，如：

    • Markdown converted to HTML
    • Shortcodes processed
    • Layout applied
    Name Markup identifiers Comment
    Goldmark md, markdown, goldmark Note that you can set the default handler of md and markdown to something else, see Configure Markup .
    Emacs Org-Mode org See go-org .
    AsciiDoc asciidocext, adoc, ad Needs Asciidoctor installed.
    RST rst Needs RST installed.
    Pandoc pandoc, pdc Needs Pandoc installed.
    HTML html, htm To be treated as a content file, with layout, shortcodes etc., it must have front matter. If not, it will be copied as-is.

    Goldmark 是 Go 语言编写的 markdown解析器，goldmark 完全符合最新的 CommonMark 规范，性能与 cmark 相当。

    在调用相应的程序处理文件时，hugo 也会默认带上一些参数，如：

    • asciidoctor : --no-header-footer -
    • rst2html : --leave-comments --initial-header-level=2
    • pandoc : --mathjax

    Front Matter

    Front matter allows you to keep metadata attached to an instance of a content type —i.e., embedded inside a content file—and is one of the many features that gives Hugo its strength.

    TOML: identified by opening and closing +++ .

    YAML: identified by opening and closing --- .

    JSON: a single JSON object surrounded by ‘ { ’ and ‘ } ’, followed by a new line.

    hugo 预设的 front matter 变量主要有：

    • date : the datetime assigned to this page.
    • description
    • draft : 布尔变量，如果为真表示当前内容为草稿，默认情况下 hugo 不会渲染该文档
    • layout
    • outputs : allows you to specify output formats specific to the content.
    • title
    • url
    • _build : build options help define how Hugo must treat a given page when building the site.
      • render : default always
      • list
      • publishResources
    • ……

    ### Hugo 中的 Markdown

    Shortcodes

    Markdown 本就是 HTML 由于变得丰富、复杂（花哨），但却不再适合直接写作后诞生的替代品。（Gruber wrote a Perl script, Markdown.pl, which converts marked-up text input to valid, well-formed XHTML or HTML and replaces angle brackets < > and ampersands & with their corresponding character entity references. ）

    书写性和丰富性中，Markdown 选择了前者，HTML 选择了后者。但作为文本丰富性的补救， markdown 支持内嵌 HTML，使得文本达到想要的表现效果。

    但内嵌 HTML 显然不优雅，也一定程度破坏了 Markdown 的简洁性， Hugo 提出采用 shortcodes 的方法规避这种限制。
